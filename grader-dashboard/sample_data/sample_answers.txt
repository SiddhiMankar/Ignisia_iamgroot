A stack is a data structure that follows the LIFO principle, meaning the last element inserted is the first one to be removed.

LIFO stands for Last In First Out. In a stack, the most recently added element is always removed first, similar to a stack of plates.

The stack data structure uses the LIFO order for insertion and deletion. Push adds an element and pop removes the top element.

In a stack, elements are added and removed from the same end called the top. This follows the last in first out order.

LIFO means the last item pushed onto the stack is the first item popped off. Stacks are used in function call management and undo operations.

A queue follows the FIFO principle — first in first out. Elements are enqueued at the rear and dequeued from the front.

FIFO means first in first out. Queues are used in scheduling and breadth-first search algorithms.

Recursion is a technique where a function calls itself with a smaller input until it reaches a base case, reducing the problem step by step.

Binary search works by repeatedly halving the search space. Its time complexity is O(log n), making it efficient for sorted arrays.

Binary search divides the array in half each time. It requires a sorted array and runs in O(log n) time complexity.
